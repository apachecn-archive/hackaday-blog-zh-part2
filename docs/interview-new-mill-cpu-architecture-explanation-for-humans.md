# 采访:为人类解释新的米尔 CPU 架构

> 原文：<https://hackaday.com/2013/11/18/interview-new-mill-cpu-architecture-explanation-for-humans/>

[https://www.youtube.com/embed/DeM-YJo3weY?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent](https://www.youtube.com/embed/DeM-YJo3weY?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent)

Hackaday 有一个很好的机会与[Ivan Godard]坐下来讨论 Mill CPU 的开发，他的公司已经进行了大约十年的开箱即用计算。Mill 开发背后的驱动力是对现有架构的优化只能让你到此为止。在某种程度上，你需要在前辈成功和失败的基础上开发出一款新的处理器。

Ivan 的团队在他们的网站上发布了几个讲座视频，深入挖掘了 Mill 相对于目前可用芯片的优势。我们[最近报道了其中一个](http://hackaday.com/2013/08/02/the-mill-cpu-architecture/)，这促使【Ivan】联系我们。但是，如果你不是在攻读半导体设计的高级学位呢？我们的采访当然不适合外行人，但任何工程爱好者都会觉得这是一次令人耳目一新、愉快的谈话。跳转之后，你可以看到四部分采访的前两部分。

如果你不想冒险看完整部电影，那就从采访中的这个最热门的片段开始吧。大约八分钟，包括三个问题。首先是讨论 Mill 如何以模仿 DSP 芯片的方式使用和预算功率，但仍然提供传统处理器的多功能性。下一个片段讨论了 Mill 对于密码分析的有用性。[Ivan]解释说，当 crypo 算法已知时， [ASIC](http://en.wikipedia.org/wiki/Application-specific_integrated_circuit) 将胜过 Mill。然而，当任务是测试发现可能用于破解密码的模式时，Mill 表现得非常出色。“最受欢迎”系列的最后一部分讨论了该团队开发的自动构建工具，这些工具将为工厂动态添加操作。在大约 20 分钟内，您可以通过模拟器看到实际操作。

不要停留在这个有趣的问题上，否则你将错过一场宏大的讨论。事实上，四部分采访的第二个片段(别担心，它们都很短)包括了我们听过的最伟大的名言之一。[Ivan]正在讨论软件初创公司和正在设计新处理器架构的公司之间的区别。他开玩笑说，软件公司可以由几个“性感”的家伙和一大笔现金创建，但是:“重型半机械不是那样工作的。重型半挂卡车就像钢铁厂和铁路。”经典！
