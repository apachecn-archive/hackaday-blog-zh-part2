# 4004 ROM 模拟器允许在慢速计算机上快速开发

> 原文:[https://hack aday . com/2012/08/01/4004-rom-emulator-allows-fast-development-on-slow-computers/](https://hackaday.com/2012/08/01/4004-rom-emulator-allows-fast-development-on-slow-computers/)

为非常旧的计算机开发是一件苦差事；你不仅会受到汇编语言或纯机器语言的限制，还会面临烧录 rom 来实际运行你的程序的问题。[Frank Buss]想出了一个为古老的 4004 CPU 开发的巧妙解决方案——使用现代微控制器构建[一个 ROM 仿真器](http://www.frank-buss.de/4004/index.html)。

构建从 4004 CPU 的 ZIF 插槽和 PIC micro 上模拟的 256 字节 4001 ROM 芯片开始。CPU 独自坐在 ZIF 插座上看起来有点孤独，所以[Frank]更新了他的主板，允许插入 4002 RAM 芯片。

因为[Frank]选择了一个 4004 作为他参加本赛季的[复古挑战赛](http://retrochallenge.org/)的参赛作品，我们需要指出的是，我们正在为向[我们的复古网站](http://retro.hackaday.com/Success/index.html)加载该 CPU 提供奖励。是的，这很可能是不可能的，但是没有什么值得做的事情是容易的。
