# 问 hack aday:Project Ara 解决了 Phonebloks 的问题吗？

> 原文:[https://hack aday . com/2013/10/30/ask-hack aday-does-project-ara-solve-the-phone bloks-problems/](https://hackaday.com/2013/10/30/ask-hackaday-does-project-ara-solve-the-phonebloks-problems/)

我们的提示线又开始膨胀了，这一次把我们引向了摩托罗拉的项目 Ara:一部带有模块化组件的手机，可以插入一个基本的“内骨骼”如果你错过了散布在网络上的新闻报道，并且你正在仔细阅读，那是因为 Project Ara 与几周前的 [Phonebloks 概念](https://phonebloks.com/)惊人地相似。Phonebloks 是我们[上一篇“问黑客日”文章](http://hackaday.com/2013/09/13/ask-hackaday-can-we-do-better-than-phonebloks/)的主题，引发了数百条评论，从捍卫这个概念的人到强烈反对它的人。

有一种阴谋论正在流传，认为摩托罗拉发布 Phonebloks 概念是一种病毒式营销计划，目的是在公布官方产品线之前进行炒作。我们怀疑这是不是有点不太纵容。正如[jorde] [在黑客新闻](http://news.ycombinator.com/item?id=6632672)上解释的那样，以色列初创公司 Modu 几年前已经开发了一款类似的模块化手机，谷歌在 2011 年 5 月购买了专利。几个月后，谷歌收购了其他公司:摩托罗拉。看起来 Project Ara 很可能只是一个复活和修改的 Modu，摩托罗拉在 Phonebloks 流行之后很方便地宣布了它。不管怎样，摩托罗拉已经宣布他们与 Phonebloks 的创造者戴夫·哈肯斯合作。

那么有什么不同呢？Phonebloks 遇到了“蒸汽器皿！”激烈的争论引发了对不可避免的硬件限制的担忧。摩托罗拉声称他们的目标是:

> 为硬件做 Android 平台为软件所做的事情:创建一个充满活力的第三方开发者生态系统，降低进入门槛，加快创新步伐，并大幅压缩开发时间表。

与 Project Ara 不同，Phonebloks [没有考虑开源硬件](https://web.archive.org/web/20130924004812/http://phonebloks.com) (Wayback Machine link)，摩托罗拉在这里提出了一个有趣的论点:3D 打印的进步表明了一个不断发展的“开放硬件生态系统”，手机开发的下一个时代可能掌握在普通黑客或小型创业公司的手中。[一些人推测](http://www.reddit.com/r/technology/comments/1pg3en/so_remember_how_we_all_had_a_good_laugh_at/cd266ks)Ara 将类似于个人电脑和其外围设备之间的关系:摩托罗拉提供必要的内脏，同时给你一些插槽来连接额外的组件。请在评论中让我们知道你对 Project Ara 的看法:它只是更多的 vaporware，还是 Phonebloks 的一个淡化但似乎可行的替代方案？或许最重要的是:作为一名黑客，你想要一部支持开放硬件并允许你插入“外设”的手机吗 Phonebloks 网站已经改变，以反映与摩托罗拉的合作关系，并包括一个新的视频，你可以在下面观看。
