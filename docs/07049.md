# THP 黑客传记:菲利克斯·鲁苏

> 原文:[https://hackaday.com/2014/09/12/thp-hacker-bio-felix-rusu/](https://hackaday.com/2014/09/12/thp-hacker-bio-felix-rusu/)

就 Hackaday 奖的参赛作品而言， [Moteino](http://hackaday.io/project/2197-The-Moteino-Framework) 格外有趣。这是唯一一个用于*黑客日奖其他项目的项目。*另外两个使用 Moteino 的项目， [433MHz 收发器](http://hackaday.io/project/2573-433MHz-Texting-and-Controller)和[工厂朋友](http://hackaday.io/project/1982-Plant-Friends)没有入选，但是【菲利克斯】的 Moteino 入选了。

像许多物联网项目一样，Moteino 是一个非常方便的无线电模块和微控制器。该收音机为 HopeRF RFM69，工作在 315、433、868 和 915MHz ISM 频段。微控制器是每个人的最爱 ATMega328，但[Felix]也有一个搭载 ATMega1284 的大型版本。已经有一些汽车旅馆能做什么的很好的例子，包括[一个邮箱通知器](http://lowpowerlab.com/blog/2013/08/27/mailbox-notifier-project-upgrade/)，一个[排水泵监控器](http://lowpowerlab.com/blog/2013/04/25/sump-pump-alert-with-moteino-hc-sr04/)，和一个[将水表互联网化的方法](http://lowpowerlab.com/blog/2013/02/02/meet-the-watermote-moteino-based-water-meter-reader-ee-sy310/)。

[费利克斯]的简历如下。

![](../Images/983edfb4479ac11c47b28dc31d15a7ba.png)

我本质上是一个创造者，所以我不得不说我喜欢创造、改进和修理我周围的东西。在我的一生中，无论是在学校还是在工作中，我都被训练以这样或那样的方式解决问题。最近，我成立了低功耗实验室，我的很多业余时间都花在了创建和博客上，描述我对互联智能家庭和环境的愿景。我也喜欢历史、护教学、摄影和其他奇怪的东西。我的妻子说她喜欢我的“烹饪”,我喜欢为她一起做这些菜:-)

![](../Images/b36bf05c59a5770a2c567b59107c46e0.png)

我白天是软件开发者，晚上是硬件黑客/制造者。当我第一次来到美国时，我在亚利桑那州立大学开始学习计算机科学，然后我想我应该尝试一些更酷的东西，比如计算机系统，因为我已经知道编程。我就是这样被介绍给硬件系统和架构的，这激发了我对电子产品的新的热爱，以及几年后在 LowPowerLab 的 [Moteino](http://lowpowerlab.com/moteino/) 驱动的冒险。

![](../Images/c4d9d090a3f9b6720f8c94bffa417268.png)

近年来，我重新找回了童年时的激情，那就是把东西拆开，然后以某种方式加以改进。我相信这是大多数黑客的共鸣主题。我曾经把我所有的玩具拆开，加上电池和用电线连接的 5V 小灯泡，做成火车“在黑暗中看见”的定制闪烁圣诞树。所以有一天我爸给我买了个共产电烙铁。就在那时，我学会了如何焊接，我对电子产品的热爱被点燃了。虽然我总是从事软件工作，但大学毕业后，我发现了 Arduino 的迷人世界，我的学位允许我开始为客户做一些小工作。焊接自然地回来了，我又爱上了电子，只是现在我是从零开始创造它们！现在我有了一个小儿子，我有一个人可以把这些东西传给他，我有一种感觉，他将会是我最大的项目。

atmega 328 p——因为它的坚固性和苦乐参半的外设集，在 Arduino 出现后，我们大多数人都变得如此熟悉。它也是我大多数项目的核心，因为它可以做大多数需要一些电子大脑的事情。

![](../Images/308a7648f6e9856896236bbb41dc80a5.png)

到目前为止，C/C++是不朽的经典之作(将来也不会)。每当你按下微波炉上的按钮或触摸汽车仪表板的显示屏时，很可能会调用一个 C 例程，一些指针传入传出；-)

![](../Images/d015faab202bc036580668d46d45464b.png)

*   嗯，完成 Moteino 框架，我感觉这永远不会真正完成
*   我想在我的社区/城市开设一门课，通过像他们自己的邮箱通知程序这样的示例项目来教孩子们电子学！
*   废除政治正确性将是一个光荣的提及

![](../Images/b695bbab69a146f980245f1ec391f5f7.png)

我已经为我的 Moteino 框架工作了一段时间，最初我不认为我的项目真的很适合 THP，因为它由许多元素组成，而不仅仅是一个特定的产品或组件。我的一个好朋友(HAD hacker Kenji Larsen)真的鼓励我进入 THP，然后我在 i3Detroit 遇到了 HAD 的人，Mike 也建议我真的应该进入，因为这是一个对各种项目都开放的地方。所以我做到了，不知何故我进入了半决赛，耶！

无论如何，不管 THP 的结果如何，Moteino 框架将继续改进和完善，以实现我的愿景，为那些喜欢 DIY 的电子爱好者提供一个可供选择的开放式设备框架。

![](../Images/26b388b1f6d5a125ea547842470c6012.png)

我喜欢坐下来和老人聊天(通常是听他们谈论他们的生活经历)。兽医尤其有趣。我总是带着一大堆令人敬畏的建议和值得学习的东西结束谈话。我们可以读书，参加会议，观看 YouTube 来学习新的技能和黑客技术，或者犯错误，然后从我们日常的制作/黑客活动中学习。但是成为一个更好的父亲、丈夫、人，来自于我意识到我不是为自己而活，而是为了让周围的人过得更好。在这方面我需要很多建议。

![](../Images/696713e81e03f54f23cda21f65a63da1.png)

我在看几个引起我注意的项目。我喜欢 [Smart Aquaponics](http://hackaday.io/project/1877-Smart-Aquaponics) 和 [Reactron Overdrive](http://hackaday.io/project/1239-Reactron-Overdrive) ，因为我对这两个主题都感兴趣，而且它们是我认为真正“连接”(通过互联网传输数据)的少数几个项目之一，也是可行和有用的。

![](../Images/1f48babecd6b5224565c529d357d07be.png)

该项目正一如既往地向前推进。每个原型制作周期大约需要 2 周时间，因为太多的事情可能会出错，所以很难为下一阶段提供更多功能的原型。但事情总是在向前发展，即使有时这意味着一脚踢在屁股上=向前两步。
