# 神经网格——是什么和不是什么

> 原文：<https://hackaday.com/2014/05/06/the-neurogrid-what-it-is-and-what-it-is-not/>

**是什么:**

有些人会说，在硅片上复制人脑是不可能的。然而，斯坦福大学硅脑实验室的人可能不同意。他们创造了一种电路板，能够实时模拟 100 万个神经元和多达 60 亿个突触。是的，那是带“B”的十亿。他们称他们的新型计算机[为神经网格](http://www.livescience.com/27907-neurogrid-brain-simulator-brain-awareness-nsf.html)。

Neurogrid board 拥有 16 个 Neurocore 芯片，每个芯片拥有 256 x 256 个“神经元”。它试图像大脑一样工作，使用模拟信号进行计算，使用数字信号进行通信。“软电线”可以在硅神经元之间运行，模仿大脑的突触。

请务必在休息后留下来，在那里我们将讨论神经网格的局限性，以及来自其创造者的视频。

**不是什么:**

尽管非常简洁和令人印象深刻，但神经网格未能解决人工智能的一个关键组成部分——软件架构。开发硬件来复制神经元的大量互连是一个很好的开始，但我们还必须从软件方面了解大脑是如何智能的，以便最大限度地利用硬件的潜力。

考虑一下计算机和模式识别的问题。人脑几乎毫不费力地做到了这一点。但是这对于计算机来说是一件非常困难的事情。通常，我们会从制作某种模板开始。然后，我们将传入的数据与模板进行比较，并基于比较做出适当的决策。如果使用 x86 机器、FPGA 或 Neurogrid 板来执行这项任务，那又有什么关系呢？将输入数据与模板进行比较的想法并不是大脑识别模式的方式。你的头脑中没有模板。

所以你认为“大脑是一台大规模并行模拟计算机，所以一旦我们可以在硬件中复制这种并行结构，我们当前的软件模型将会工作得更好。”

也许吧。但是考虑下面的类比。你有 100 块石头需要移到沙漠的另一边。穿越沙漠需要一百万步。所以你雇了一百个人来搬运石头穿过沙漠。雇佣超过 100 人也不会让石头更快地穿过沙漠。你还是要走一百万步。*

总有一天，简单地增加更多的并行流程不再等同于提高效率。我们应该问自己是否有其他穿越沙漠的方法，而不是试图让老方法(步行)更有效。

*从杰夫·霍金斯的《论智力的》中借用的平行类比。
