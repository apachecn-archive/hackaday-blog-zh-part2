# VCF 东方 X:没用的八十年代

> 原文：<https://hackaday.com/2015/04/18/vcf-east-x-the-not-trashy-eighty/>

地位低下的 TRS-80 在大多数圈子里得不到多少爱；它不断被苹果 II 或图形并不糟糕的电脑的流行所掩盖。对于[迈克·勒文]的 VCF 展览，[他正在用 SD 卡磁盘驱动器和定制图形适配器把他的 TRS-80 变成好东西](https://www.youtube.com/watch?v=Q6ndUYkEfSk&feature=youtu.be)。

问题中的-80 是 Model 4，可以运行 CP/M 的花哨的一体化版本。该计算机中的磁盘驱动器被替换为半高 5 1/4”驱动器，200 纳秒 RAM 被替换为 100 纳秒 RAM，并经过修改以消除等待状态，硬盘驱动器在 SD 卡适配器[上模拟，这要归功于来自[Ian Mavric]的插件](http://ianmav.customer.netspace.net.au/trs80/)。

[伊恩]在 TRS 80 年代的世界里是多产的；他对最初的高分辨率显卡进行了逆向工程，并用更现代的视频 RAM 芯片重新实现了它。
