# 演示 An 8088

> 原文：<https://hackaday.com/2015/04/10/demoing-an-8088/>

演示场景通常围绕 Commodore 64 展开，当您将 C64 硬件与类似年份的其他计算机进行比较时，很容易看出原因。在一个芯片上有一个完整的三声音合成器，硬件允许精灵，大量的视频页面，还有令人震惊的 16 种颜色，其中大多数看起来很好。你不会找到很多 Apple II 的演示，因为图像和声音都很糟糕。你也找不到 1981 年的 IBM 个人电脑的演示，因为三十年来，图形和音频一直很糟糕。

由[大黄蜂]、[CRTC]和[欲望]设计的 8088 英里/小时，最近 2015 年修订版演示 compo 的获胜者刚刚颠覆了传统观念。它运行在 4.77 MHz 的 8088 CPU 上，与最初的 IBM PC 相同。图形是由一种特殊的 IBM CGA 卡通过复合输出提供的，声音是个人电脑扬声器的哔哔声，每秒钟哔哔声六十次。[这里有一段视频截图](https://www.youtube.com/watch?v=yHXx3orN35Y)。

由于这个演示的极端性质，它不能在任何模拟器上运行。虽然最初的开发是在装有 DOSbox 的现代机器上进行的，但完成演示需要在 IBM 5160 上进行，相当于 5150，但更容易找到。

尽管硬件简陋，CPU 在四个周期内读取一个字节，有效地使其成为 1.19 MHz 的 CPU，但该团队制作了所有常见的演示场景视觉效果。有莫尔图案、跳动的文本、旋转和缩放的位图，以及令人惊讶的 1024 色模式，这是对 NTSC colorburst 打开时 80×25 文本模式的惊人滥用。
