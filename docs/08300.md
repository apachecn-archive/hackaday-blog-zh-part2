# Raspi 上的图文电视，没有附加部件

> 原文：<https://hackaday.com/2015/02/13/teletext-on-a-raspi-with-zero-additional-parts/>

早在 70 年代，英国和英国广播公司就推出了图文电视——一种信息检索服务，比互联网和万维网更接近于 20 世纪 60 年代未来主义者流行的“连接到房间大小的计算机的电视屏幕”。在大约 30 年的时间里，图文电视是最可靠的信息传播手段之一，直到它随着数字电视的推出而被悄悄搁置。

不过，玩死协议很有趣，因为 Raspberry Pi 有模拟视频输出，[Alistair]认为将他的 Pi 变成图文电视生成器并显示会很有趣。

这不是[阿利斯泰尔]的第一次图文电视竞技；早些时候，他为 Raspi 构建了一个附加板[，它使用 AVR 和 LM1881 视频同步分离器将 Raspi 的视频输出与图文电视信号进行多路复用。新版本完全摒弃了这一点，允许任何 Raspberry Pi 生成并显示来自文字电视广播服务的信息。现在有两个演示，一个 Raspi 状态显示，显示 CPU 频率，使用，内存和温度。还有一个“时钟破解程序”，上面有一张燕尾服的照片，应该有助于诊断接待问题。](http://hackaday.com/2013/07/07/raspberry-pi-learns-the-lost-art-of-teletext/)

所有代码都可以在该项目的 github 上获得[，尽管【Alistair】还没有发布脚本来输出几年前从广播信号中捕获的图文电视页面。](https://github.com/ali1234/raspi-teletext)
