# 本周失败:从激光测距仪获取数据

> 原文:[https://hacka day . com/2013/09/26/从激光测距仪获取周失败数据/](https://hackaday.com/2013/09/26/fail-of-the-week-capturing-data-from-a-laser-rangefinder/)

![fotw-laser-measuring-tape](../Images/9dce249f8f8a8863e382f64bb6532c0f.png)

本周，我们通过一个逆向工程的失败改变了它，[意大利]向我们指出了这一点。几年前，Sparkfun 的[Nate]同意帮助一位朋友完成一个需要精确测量距离的项目。他知道激光测距仪是一种很好的方法，并提到了它们在高尔夫和建筑行业的应用。他拿起了这个被宣传为激光卷尺的手持版本。他勇敢地对 PCB 进行逆向工程，希望找到一个测量数据的挂钩。

很明显他的努力失败了，否则我们不会在这个专栏里谈论它。但是关于他的方法还有很多需要学习的地方，一些与他的原始帖子相关的评论有助于阐明一些额外的尝试。

首先，这东西不便宜——至少要 90 美元。这不是一件坏事，因为电子设备都是精心设计的。[Nate]认为电路板上有某种类型的调试信息是正确的，他必须找到它。在用连续性测试仪测试了一段时间后，他能够弄清楚上面显示的测试焊盘的用途，并在上面焊接了一些引脚，以便连接他的测试设备。

![Tags2-M](../Images/346eb8d4582e827c37d7729392258739.png)

那么下一步是什么？使用逻辑嗅探器是逆向工程过程中的下一个逻辑步骤。[Nate]进行了大量的捕获，最终发现了一小段串行数据。他使用的软件识别出它是 19543 波特，这与 19200 标准足够接近，可以称之为匹配。但进一步努力发送或接收超过？\n \r '设备输出失败。

这是怎么回事？已经在原始线程上发布的一些想法包括需要将线路接地以使芯片说话、流量控制问题、尝试使用 at 命令，以及在编程期间烧毁 RX 线路上的保险丝以防止篡改/读取固件的可能性。我们当然很想听听你下面的想法！
