# 矢量图形 Flappy 鸟比它应该更难

> 原文：<https://hackaday.com/2014/05/26/flappy-bird-mon-am/>

Maker Faire 的暗室很吵，毕竟 Arc Attack 就是在这里成立的，还有其他几个有音乐的显示器。但是，如果你勇敢地面对音频，并设法不因为眨眼而经历癫痫发作或偏头痛，你会在 TubeTime 展台的展览中看到这些[急剧发光的矢量显示器。我们尽了最大努力拍摄，但如果亲自观看，线条的清晰度以及磷光图像与黑屏的对比度似乎仍然更突出。](https://www.youtube.com/watch?v=mtHzBT0rmG8)

[https://www.youtube.com/embed/mtHzBT0rmG8?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent](https://www.youtube.com/embed/mtHzBT0rmG8?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent)

这不是埃里克第一次尝试驱动高压显像管显示器。我们之前报道过[他的 dekatron 厨房定时器](http://hackaday.com/2009/04/06/dekatron-kitchen-timer/)。但我们要说，从那时到现在的几年里，他确实把事情提高了几个档次。他[写了一篇关于小行星](http://tubetime.us/?p=173)的博客，它运行在与我们上面视频中的 Flappy Bird 演示相同的硬件上。一个 STM32F4 发现板正在运行 6502 仿真器，将游戏推送到[Eric 的] CRT 矢量驱动硬件。

就在我们在展台结束之前，[埃里克]转过身来，眼睛闪闪发光。他承认自己喜欢故意省略 Flappy Bird 演示中的任何按钮去抖。就好像它还不够难一样，在通过几个管道门之后，它就开始出故障了。穆呼哈哈！
