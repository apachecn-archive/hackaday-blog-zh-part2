# 弗兰肯斯坦，开源引擎控制单元

> 原文：<https://hackaday.com/2014/03/29/frankenstein-the-open-source-engine-control-unit/>

发动机控制单元是过去 40 年左右制造的每辆汽车的重要组成部分，但与几乎所有其他电子设备不同的是，开源解决方案根本不存在。[Andrey]正试图通过 ruse fi 改变这种情况，该项目希望将硬件、软件和引擎整合到一个易于使用的包中。他甚至为 STM32F4 发现开发板设计了一个完整的 ECU“盾牌”[弗兰肯斯坦](http://rusefi.com/forum/viewtopic.php?f=4&t=359)。

这不是我们第一次看到[Andrey]建造 ECU 的冒险经历。[更早的电路板](http://hackaday.com/2014/01/01/building-an-engine-control-unit-with-the-stm32f4/)也由 STM32F4 Discovery 驱动，他实际上驾驶着他的 96 款福特 Aspire 到处跑，使用的就是这种自制的 ECU。它只是在两个气缸上点火，但那只是一个松散的焊接连接。

当然，如果没有适当的固件来平衡发动机的燃油经济性和性能，从头开始构建 ECU 是没有价值的。这种测试必须凭经验进行，而且[Andrey] [有一个 Kickstarter 去](https://www.kickstarter.com/projects/312898525/rusefi-gpl-automotive-engine-control-unit-ecu-firm)开发这个固件和一些 dyno 时间。没有奖励，但花一两块钱是值得的。我做到了。
