# 数百万个卫星接收器对僵尸网络来说是唾手可得的

> 原文:[https://hack aday . com/2015/08/09/百万卫星接收器是僵尸网络的唾手可得的果实/](https://hackaday.com/2015/08/09/millions-of-satellite-receivers-are-low-hanging-fruit-for-botnets/)

卫星电视在欧洲和北非很流行。这是通过机顶盒(STB)传送的，机顶盒使用读卡器来解码加扰的卫星信号。想看的话需要买卡。但是你知道人们喜欢不劳而获。黑客正在利用这一点，结果是数百万个机顶盒正等着形成僵尸网络。

这是在 DEF CON 23 上[Sofiane tal mat]演讲的主题。本周早些时候，他也在 BlackHat 做过这个演讲，[已经发布了他的幻灯片](https://www.google.com/url?sa=t&rct=j&q=&esrc=s&source=web&cd=2&cad=rja&uact=8&ved=0CEIQFjABahUKEwiCz5TF7ZrHAhWDL4gKHSWUAHw&url=https%3A%2F%2Fwww.blackhat.com%2Fdocs%2Fus-15%2Fmaterials%2Fus-15-Talmat-Subverting-Satellite-Receivers-For-Botnet-And-Profit.pdf&ei=vq3GVYL4MoPfoASlqILgBw&usg=AFQjCNF9qlO16UTYfsRgTzMl63Ul6eytjA&bvm=bv.99804247,d.cGU) (PDF)。

卫星接收机中的硬件运行的是 Linux。他们使用一个读卡器来读取一个码字(CW ),这个码字对通过卫星无线电接收到的信号进行解码。

围绕这些暗语已经形成了一个完整的黑市。人们不再购买有效的卡，而是从互联网上安装插件，使系统连接到一个提供有效代码字的服务器。这就是所谓的“卡共享”。

在用户方面，这是可行的；用户免费看电视。这可能会导致比正常情况下更多的崩溃，但股票软件是有缺陷的，所以这不是一个重大的回归。问题是，现在这些人已经将一个联网的 Linux 盒子暴露在互联网上，并安装了来自未知来源的未经验证的代码来运行这个东西。

[Sofiane]展示了创建僵尸网络只需了解该系统的一点点内容:

*   用 C/C++构建一个插件
*   托管卡共享服务器
*   僵尸网络受害者来找你(盈利)

事实上就是这么简单。编译 STLinux 二进制文件(gcc)的工具链可以在 Linux repos 中找到。STB 将在启动时在 USB 拇指驱动器上寻找一个“bin”目录，该文件夹中的二进制文件将被自动安装。由于用户得到了免费电视，他们自愿安装这个恶意软件。

这里有一个主要的例子，说明了为什么当你下载软件安装到你自己的系统上时，你总是想要验证校验和。[Sofaine]在互联网上的许多下载网站上研究了用于卡共享的“相同”软件包，并且有多个不同的校验和。假设这些病毒携带不同的恶意软件载荷。

除了这种简单的利用方式之外，无论如何，盒子都是被设计破坏的。没有防火墙，有二级根帐户(后门)，有默认运行的 FTP 服务器，有根权限，没有密码。对我来说，最可笑的漏洞是制造商的更新并没有对操作系统做任何修补或改进，它们是 100%的用户体验更新。在演示机器上运行的 BusyBox 版本来自 2012 年，有多个已知的漏洞。即使你不想使用卡共享服务，只要连接到互联网，设备就可能受到威胁。

这个演讲是在村里进行的，不是在主舞台上。这是一个很好的例子，说明为什么你应该认真对待这些谈话。你将会得到比在高度抛光的“轨道”会谈上更坚韧不拔的解释和示范。你也有机会提问，人们不太可能只是为了听自己说话而提问(这种情况在这里发生得太多了)。
